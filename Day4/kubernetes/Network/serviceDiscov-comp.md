Kubernetes Network and Service Discovery Components
Service Discovery is the mechanism by which applications (like microservices) in a distributed system find and communicate with each other dynamically without needing to know the ephemeral (short-lived) IP addresses of their peers. In Kubernetes, where Pods (and their IPs) are constantly created, destroyed, and scaled, Service Discovery is essential for reliable communication. Services act as a stable middle layer.

It abstracts away the dynamic nature of Pod IP addresses.ComponentDescriptionRole in Service DiscoveryService ObjectA permanent, named abstraction over a group of Pods, selected via labels and selectors.Provides a stable ClusterIP and DNS name that clients can use, regardless of which Pods are running behind it.ClusterIPThe default Service type. An internal, stable IP address reachable only from within the cluster.Used for internal microservice communication and acts as a virtual IP for the Service.NodePortExposes the Service on a static port on every Node's IP.Allows external traffic to reach the service via <NodeIP>:<NodePort>.LoadBalancerExposes the Service externally using a cloud provider's load balancer.The standard way to provide public external access, automatically provisioning and configuring the necessary cloud resource.ExternalNameMaps the Service to an external DNS name (using a CNAME record).Used for accessing services outside the cluster using a consistent name within the cluster.Headless ServiceA Service with no ClusterIP.Used when you need to interact directly with the individual Pods' IPs (e.g., for StatefulSets or custom discovery logic). DNS (The Discovery Mechanism)Kubernetes includes a cluster-wide DNS server (usually CoreDNS) that automatically creates DNS records for every Service.ComponentDescriptionRole in Service DiscoveryCoreDNS / kube-dnsThe internal DNS server running as Pods in the cluster.It resolves a Service's DNS name (e.g., my-service.my-namespace.svc.cluster.local) to its ClusterIP or, for Headless Services, to the IPs of the backend Pods.
